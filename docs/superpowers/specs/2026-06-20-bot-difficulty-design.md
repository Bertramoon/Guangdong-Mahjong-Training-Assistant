# 机器人出牌难度分级设计

## 背景

机器人出牌原本只有两档，由布尔开关 `robotSmartDiscard` 控制：

- **关闭（启发式）**：保鬼牌 → 先弃孤张字牌 → 再弃孤张数牌 → 弃对子一张 → 最后随机。偏弱，像一个"保对子、丢孤张"的新手。
- **开启（智能）**：取向听数推荐排序的 TOP1。太强——三个机器人合计赢面远大于单个玩家，玩家难以获胜。

缺少中间难度。

## 方案：5 档难度 + 失误率混合

把单一布尔开关升级为 5 档枚举难度，每档对应"本回合走智能出牌"的概率 p：

| 难度 | p（走智能） | 含义 |
|------|-----------|------|
| 关闭 off | 0.0 | 纯启发式（等价旧"关闭"） |
| 低 low | 0.2 | 多数回合启发式，偶尔智能 |
| 中 medium | 0.6 | |
| 高 high | 0.85 | |
| 最高 master | 1.0 | 恒智能（等价旧"开启"） |

每回合按当前难度的概率独立掷骰：命中则走智能出牌（向听数推荐 TOP1，经 Web Worker 计算），否则回落到现有启发式"失误手"。这样机器人"偶尔手滑"、整体建牌更慢，给玩家追赶窗口，又不会像纯随机那样明显摆烂。

相比"在 TOP-N 推荐里随机选一档"的方案，概率混合在中盘也能稳定弱化（启发式不优化向听数，而 TOP-N 方案在中盘大量候选牌向听数相同、随机选与选最优常常等价），更对症"玩家赢不了"。

## 设计落点

- 策略切换集中在 `useGame.ts` 的 `chooseRobotDiscard`：每回合掷骰决定走智能还是启发式。
- 难度枚举、概率表、以及供设置迁移用的纯函数集中在 `src/robot/difficulty.ts`。
- 设置项 `robotDifficulty` 取代旧 `robotSmartDiscard`，默认"关闭"；读取旧设置时自动迁移（旧 `true`→最高、`false`→关闭）。
- 设置面板用下拉框选择 5 档。

## 范围

- 仅影响"自己回合主动出牌"；碰/杠反应保持不变。
- 全局难度：3 个机器人同一档位（不做每机器人差异化）。
- 失误手沿用现有启发式（不改为"从排名靠后候选里选"）。
