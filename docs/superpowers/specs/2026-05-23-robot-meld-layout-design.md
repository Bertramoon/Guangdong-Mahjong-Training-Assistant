# 机器人副露布局优化设计

## 问题描述

当前采用响应式布局，中间行使用 `justify-content: space-between`。当左右两侧机器人（西、东）的副露（碰、杠）较多时，副露区域水平展开，严重挤占中间弃牌池的可用宽度。弃牌池被迫变窄后只能纵向延伸，导致单页无法容纳全部信息。

## 目标

- 保证弃牌池有足够的最小显示宽度，不被两侧副露挤压。
- 在副露较多时，左右机器人的副露区域从水平展开改为垂直堆叠，减少水平空间占用。
- 保持顶部机器人（北）和玩家自身（南）的副露展示方式不变。

## 方案

### 1. MeldArea.vue — 新增排列方向控制

- 新增 `direction` prop，类型为 `'row' | 'column'`，默认 `'row'`。
- 当 `direction` 为 `'column'` 时，`.meld-area` 的 `flex-direction` 设为 `column`。
- 垂直模式下将 `.meld-area` 的 `gap` 从 `12px` 减小到 `6px`，使堆叠更紧凑。
- 每个 `.meld-group` 内部保持横向排列（即每张牌仍水平展示），不受影响。

### 2. OtherPlayer.vue — 根据位置选择方向

- `position` 为 `'left'` 或 `'right'` 时，向内部 `MeldArea` 传递 `direction="column"`。
- `position` 为 `'top'` 时保持默认 `direction="row"`（顶部副露不挤压中间宽度，无需改动）。

### 3. DiscardPool.vue — 弃牌池最小宽度保护

- 将 `.discard-grid` 的 `max-width: 320px` 移除，新增 `min-width: 280px`。
- 确保弃牌池在任何情况下都不会被挤压到小于 `280px`，同时在大屏幕上仍可响应式扩展。

### 4. GameBoard.vue — 中间区域兜底保护

- 给 `.board-center` 增加 `min-width: 300px`，作为极端情况下的兜底保护，确保中间区域不被完全压没。

## 预期效果

- 左右副露垂直堆叠后，各自的水平宽度从 `200px+` 缩减到约 `80-100px`。
- 中间区域获得充足宽度，弃牌池在 `min-width: 280px` 下每行可稳定容纳约 8 张牌，整体高度可控。
- 即使某个机器人有 3-4 个副露，垂直堆叠的总高度（约 `150-200px`）仍在中间行的合理范围内。

## 边界情况

- **副露极少时（0-1个）**：垂直堆叠与横向排列视觉效果差异不大，不影响可读性。
- **副露极多时（4个）**：垂直堆叠会占用较大高度，但不会超出中间行的合理范围。若需进一步保护，可后续考虑给 `other-melds` 增加 `max-height` + 滚动，但本次不做。
- **小屏幕设备**：`min-width: 280px` 配合 `board-center` 的兜底保护，可确保布局不会崩掉。若屏幕确实过小，整体页面会出现横向滚动，这是可接受的降级方案。
