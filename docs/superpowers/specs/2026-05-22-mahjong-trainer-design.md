# 广东麻将训练助手 - 设计文档

## 概述

一个网页端的广东麻将训练工具，帮助玩家通过实战对局提升麻将技术。1 个真人玩家 + 3 个机器人，每步操作可调用大模型分析，提供最佳建议和理由。

## 产品定位

- **核心目标**：在实际对局中帮助玩家理解每一步的最优策略
- **用户场景**：单人使用，无需联网后端，浏览器打开即玩
- **训练价值**：AI 实时分析让玩家建立牌感，理解出牌逻辑

## 技术选型

| 层面 | 选择 | 理由 |
|------|------|------|
| 框架 | Vue 3 + TypeScript | 用户偏好，中文社区活跃 |
| 构建 | Vite | 快速开发，Vue 生态首选 |
| 数据存储 | localStorage | 纯前端，无后端依赖 |
| AI 接口 | OpenAI 兼容格式 | 覆盖主流供应商 |
| 样式 | CSS/SCSS | 麻将牌面自定义渲染 |

## 模块详细设计

### 1. Game Engine（牌局核心引擎）

**职责**：管理麻将牌局的所有规则和状态，与 UI 完全解耦。

**牌库**：136 张
- 万子：一万~九万，各 4 张 = 36 张
- 条子：一条~九条，各 4 张 = 36 张
- 筒子：一筒~九筒，各 4 张 = 36 张
- 风牌：东南西北，各 4 张 = 16 张
- 箭牌：中发白，各 4 张 = 12 张

**鬼牌机制**：
- 开局随机翻出一张牌作为鬼牌指示
- 与指示牌同类的所有牌均为鬼牌
- 鬼牌不可碰杠，可代替任意牌组成胡牌
- 胡牌判定需考虑鬼牌的万能替代

**核心数据结构**：

```typescript
// 牌的类型
type TileType = 'wan' | 'tiao' | 'tong' | 'feng' | 'jian';
type Tile = { type: TileType; value: number; id: number };

// 副露类型
type MeldType = 'peng' | 'ming_gang' | 'an_gang' | 'jia_gang';
type Meld = { type: MeldType; tiles: Tile[] };

// 玩家状态
type PlayerState = {
  hand: Tile[];
  melds: Meld[];
  discards: Tile[];
  isRobot: boolean;
};

// 游戏阶段
type GamePhase = 'deal' | 'draw' | 'discard' | 'peng' | 'gang' | 'hu' | 'end';
type GameState = {
  wall: Tile[];
  players: PlayerState[4];
  currentPlayer: number;       // 0-3
  phase: GamePhase;
  ghostTile: TileType & { value: number };  // 鬼牌指示
  discardPool: Tile[];         // 当前轮出牌
  turnHistory: TurnRecord[];
};
```

**推倒胡规则**：
- 不能吃牌，只能碰/杠/胡
- 只支持自摸
- 胡牌判定：标准 3n+2 模型（若干面子 + 一个雀头）
- 含鬼牌的胡牌判定需递归尝试替代

**游戏流程**：
```
开始 → 洗牌 → 每人13张 → 庄家14张 → 翻鬼牌
  → 当前玩家摸牌 → 检查自摸胡/暗杠
  → 出牌 → 其他玩家检查碰/杠
  → 下一个玩家 → 循环
  → 牌墙空或有人胡牌 → 结束
```

### 2. Robot Player（机器人策略）

**职责**：模拟 3 个机器人玩家的行为。

**出牌逻辑**：
1. 分析手牌中的单张（不成对、不成搭的孤张）
2. 从单张中随机选一张丢弃，优先丢弃字牌孤张
3. 没有单张时，从双张中随机选一张丢弃
4. 不做复杂的拆搭和进张分析

**碰牌**：
- 当有人打出一张牌，机器人手中有两张相同的牌时，自动碰牌

**杠牌**：
- **明杠**：别人打出第四张相同牌时，自动杠
- **暗杠**：自己摸到第四张相同牌时，自动暗杠
- 杠后从牌墙末尾补一张牌

**不胡牌**：机器人永远不会宣告胡牌，游戏只有玩家能赢或流局

**不做策略优化**：保持简单随机行为，突出训练工具定位

### 3. AI Analyzer（大模型分析模块）

**职责**：调用大模型分析当前牌面，给出建议和理由。

**多模型适配**：
- 统一接口 `AIProvider`，支持 OpenAI / Claude / DeepSeek / Ollama 等
- 用户在设置页配置：API Endpoint、API Key、Model Name
- 使用 OpenAI 兼容格式（`/v1/chat/completions`）作为标准接口

**Prompt 结构**：
- 当前手牌（排序后的完整列表）
- 当前副露（碰/杠的组合）
- 所有弃牌池的牌（四家已出牌）
- 其他玩家的副露
- 当前鬼牌
- 牌墙剩余数量
- 当前可选操作（出牌/碰/杠/胡）
- 要求 AI 以 JSON 格式返回：`{ recommendation, reasoning, alternative }`

**触发机制**：

- **自动模式**：每次轮到玩家操作时自动请求分析
- **手动模式**：显示"分析"按钮，点击后请求
- 用户可在设置中切换模式

**性能**：
- 显示加载状态
- 分析请求与游戏操作独立，不互相阻塞
- 缓存上一次分析结果

### 4. UI Layer（界面层）

**整体布局**：
```
┌─────────────────────────────────────┐
│           对面玩家 (北)              │
│         [副露] [手牌背]             │
│                                     │
│  左(西)  ┌───────────┐  右(东)      │
│ [副露]   │  弃牌池   │  [副露]      │
│ [手牌背] │           │  [手牌背]    │
│          └───────────┘              │
│         [副露] [你的手牌]           │
│           你的位置 (南)              │
│    [分析面板]  [操作按钮]           │
└─────────────────────────────────────┘
```

**核心组件**：
- `GameBoard`：主游戏桌面，包含四方位布局
- `PlayerHand`：手牌展示（自己的牌正面，其他玩家背面）
- `MeldArea`：副露区域（碰/杠展示）
- `DiscardPool`：中央弃牌池
- `ActionPanel`：操作按钮（碰/杠/胡/过）
- `AIAnalysisPanel`：AI 建议面板
- `SettingsModal`：设置弹窗（AI 配置、模式切换）
- `GameHistory`：牌局记录

**交互设计**：
- 点击手牌选中，再点击出牌区或按"出牌"按钮完成出牌
- 碰/杠/胡机会出现时弹出操作按钮
- **无倒计时**：玩家操作不受时间限制，可以慢慢思考和查看 AI 建议
- 机器人之间自动快速流转
- AI 分析结果以卡片形式展示，显示建议牌和理由

**视觉风格**：
- 绿色麻将桌面背景
- 牌面使用 SVG 渲染，清晰可辨
- 响应式设计，支持桌面和平板

### 5. Storage（数据存储）

- 使用 localStorage 存储游戏设置和 AI 配置
- 可选：存储牌局历史记录用于回顾

## 关键算法

### 胡牌判定（含鬼牌）

核心难点在于鬼牌的万能替代。算法思路：
1. 统计手牌中鬼牌数量
2. 递归尝试：将鬼牌替代为每种可能的牌，检查是否能构成 3n+2 的标准胡牌型
3. 优化：先去除已有的完整面子，减少搜索空间

### 牌效分析（为 AI Prompt 准备）

计算当前手牌的进张数（等待哪些牌能改善牌型），作为 AI 分析的辅助信息。

## 界面流程

1. **进入页面** → 显示主菜单（开始新局/设置/历史记录）
2. **设置** → 配置 AI 供应商、API Key、触发模式
3. **开始新局** → 洗牌动画 → 发牌 → 翻鬼牌
4. **对局中** → 轮到玩家时可选查看 AI 分析 → 操作
5. **局结束** → 显示结果（胡牌/流局）→ 可查看本局回顾 → 返回主菜单
